项目名称：《人民日报》综合分析系统

数据来源：人民日报图文数据库
主要内容：爬取2021年4月人民日报的全部新闻包括：日期，页码，标签，标题，链接，内容等要素存到本地sql中，用Python Flask搭建网站框架，用css对前端页面进行美化，使用d3.js进行数据可视化，构建了包括《人民日报》四月新闻速览、5个专栏（要闻、奋斗百年路 启航新征程、国际、社会、法治）主题-关键词发展趋势图和详情展示、专栏-关键词网络分析和知识图谱三个方面的综合新闻分析系统，并计算了各关键词节点的出入度和PageRank、Centrality中心性。

各页面说明：各页面均存放在ui文件夹下的template文件夹，顶部栏（header.html）、侧边导航栏(sidebar.html)和基本模板（base.html）放在了common文件夹中作为基本模板在每个页面继承。

（1）通过/quicklook路由进入《人民日报》速览（qucklook.html）页面，本页面主要展示了所有新闻的标题、内容，并对新闻内容进行关键词提取（词频统计）和情感分析（SnowNLP），用jieba分词后根据词频高低绘制全部内容的词云图（d3.js），直观的展示了4月《人民日报》全部新闻的主要内容。

（2）通过/keynews路由进入《人民日报》要闻专栏(keynew.html)，主要界面左右分页，左面是要闻专栏五大主题的主题-关键词发展趋势图，右侧是相关新闻的具体详情，新闻内容可折叠（只展示标题）。同理，/theme路由下的奋斗百年路 启航新征程专栏（theme.html），/global路由下的国际新闻专栏（global.html），/social路由下的社会新闻专栏（social.html），/laws路由下的法制新闻专栏（laws.html）也是如此。

（3）通过/net路由进入《人民日报》专栏-关键词分析得出的知识图谱（net.html）,用d3.js制作了力导向图，直观展示了节点之间的连接关系，并通过搜索框可以搜索某一关键节点相关节点的出入度、中心性等具体信息，右侧则是所有节点的出入度统计图和PageRank、Centrality的极值点信息。
动态主题模型：代码是单独放置的，DTM_rmrb.ipynb
sql文件：rmrb.sql
分析结果说明：
（1）主题-关键词动态发展趋势图：位于每个专栏网页的页面左侧，基于动态主题模型（Dynamic Topic Model）实现。每张图片代表所处专栏下的一个主题，图片中绘制的是该主题下各关键词所占比重随时间变化的折线图，以及随时间变化是否有新的关键词产生或旧的关键词消逝。
 （2）专栏-关键词知识图谱：基于动态主题模型分析出的  关键词->主题->专栏  网络生成。其中，PageRank值可理解为“指向该节点的向量最多的节点，即主题丰富度最高的专栏”，前三位分别为“要闻栏目”、“国际栏目”和“法治栏目”；Centrality值可理解为“建立关系最多的节点，即4月份报道中提及最多的主题”，前三位分别为“医疗体制改革”、“民生保障”和“企业服务”。
                                           结合实际来看，人民日报每日版面基本都会有“要闻栏目”，且该栏目中包含的是当日最重要的新闻内容，直接指向它的主题虽然只有5个，但每个主题所包含的关键词节点范围很广，则其成为PageRank最高的节点是符合实际的；从Centrality值的计算来看，人民日报4月份聚焦的前三位主题无一不与“民生”有关，可以看出当前反复强调的仍然是如何保障民生、促进发展方面的政策。
